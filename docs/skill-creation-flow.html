Skill Creation Flow

From scaffold to packaged skill — the complete development lifecycle

1
Scaffold
Generate the skill directory structure from templates
⌄

Command

PYTHONPATH=skills/adk-skill-creator \
python skills/adk-skill-creator/scripts/scaffold.py \
  <name> <output_dir>

What you get

  • <name>/SKILL.md — frontmatter + instructions template
  • <name>/tools.py — SkillRuntime class skeleton
  • <name>/evals/evals.json — eval cases template
  • <name>/evals/eval_factory.py — mock runtime template
2
Edit SKILL.md
Write the description that controls activation and the instructions body
⌄

File

<skill_dir>/SKILL.md

What to fill in

  • description — precise natural-language trigger; this is what the agent reads to decide whether to activate the skill
  • adk_additional_tools — list of SkillRuntime method names to expose as tools (must match tools.py exactly)
  • Instructions body — step-by-step guidance for the agent operating inside this skill

Key rule

The description field is the sole activation signal — write it to match real user prompts, not internal concepts.

3
Edit tools.py
Implement SkillRuntime with real dependencies and tool methods
⌄

File

<skill_dir>/tools.py

What to implement

  • SkillRuntime.__init__ — inject real dependencies (API clients, DB handles, config)
  • Tool methods — one public method per tool; each becomes a callable the agent can invoke
  • get_tools() — return list of method references to expose

Key rule

Method names in get_tools() must exactly match the adk_additional_tools list in SKILL.md — step 4 catches any mismatch.

4
Validate
Check that SKILL.md and tools.py are in sync before running evals
⌄

Command

PYTHONPATH=skills/adk-skill-creator \
python skills/adk-skill-creator/scripts/quick_validate.py \
  <skill_dir>

What it checks

  • SKILL.md exists and has valid frontmatter
  • tools.py defines a SkillRuntime class
  • Every name in adk_additional_tools is a public SkillRuntime method
  • No undeclared public methods exist in SkillRuntime

What you get

List of mismatches to fix, or a clean exit. Must pass before running evals.

5
Write Evals
Define realistic user prompts, expected behavior, and a mock runtime
⌄

Files to edit

  • <skill_dir>/evals/evals.json
  • <skill_dir>/evals/eval_factory.py

evals.json — each eval needs

  • prompt — a realistic user message (not a synthetic one)
  • expected_skill_activated — true or false
  • expectations — list of verifiable assertions about the response

eval_factory.py

Return a mock SkillRuntime with fixture data so evals run without real backend calls.

6
Run Evals
Execute prompts against Gemini and capture transcripts + metrics
⌄

Command

PYTHONPATH=skills/adk-skill-creator \
python skills/adk-skill-creator/scripts/run_eval.py \
  <skill_dir> \
  --model gemini-2.0-flash \
  [--project <gcp-project>] \
  [--location us-central1]

What you get

  • runs/<timestamp>/eval_N/outputs/transcript.md
  • runs/<timestamp>/eval_N/outputs/metrics.json
  • runs/<timestamp>/eval_N/outputs/skill_activation.json
  • runs/<timestamp>/eval_N/timing.json
7
Grade
Run the grader agent on each eval output to score assertions
⌄

How

Run the agents/grader.md agent on each runs/<timestamp>/eval_N/ directory.

What the grader checks

  • Skill activation — deterministic, reads skill_activation.json
  • Each expectation in evals.json — LLM-judged pass/fail
  • Tool call sequence correctness

What you get

Pass/fail per assertion. Use agents/comparator.md to compare two skill versions head-to-head.

↩

Feedback Loop

Based on grader output, route to the appropriate fix and re-run from that step

🔴 Skill not activating — skill_activated: false on cases that should trigger → Step 8 — Optimize description
🟡 Tool calls wrong — wrong method called or wrong arguments → Step 3 — Edit tools.py
🟠 Expectations fail — response content or format is wrong → Step 2 — Edit SKILL.md
8
Optimize Description
Test candidate descriptions against trigger evals until activation accuracy is satisfactory
if not activating
⌄

First, create a trigger evals file

[
  {"query": "show my recent transactions", "should_trigger": true},
  {"query": "what is the weather today",   "should_trigger": false},
  {"query": "what did I spend last week",  "should_trigger": true}
]

Then test a candidate description

PYTHONPATH=skills/adk-skill-creator \
python skills/adk-skill-creator/scripts/run_loop.py \
  <skill_dir> trigger_evals.json \
  --description "Your candidate description here"

What you get

Accuracy score + per-query results. Refine the description based on failing cases and repeat. When satisfied, update description in SKILL.md and re-run evals from step 6.

9
Package
Validate and bundle the skill into a distributable .skill zip
for distribution
⌄

Command

PYTHONPATH=skills/adk-skill-creator \
python skills/adk-skill-creator/scripts/package_skill.py \
  <skill_dir>

What you get

  • <skill_dir>.skill — validated zip ready for the skill registry
